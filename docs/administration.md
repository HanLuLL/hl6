# 管理后台配置

本文面向 HL6 管理员，按功能域说明后台配置、依赖关系和操作风险。环境变量、容器和反向代理由部署人员维护，见[生产部署与升级](deployment.md)。

## 1. 管理员与权限

首个完成 OIDC 注册的用户自动成为管理员。后续管理员身份满足任一条件即可：

- 用户 `role` 为 `admin`。
- 用户所属用户组启用了管理员权限。

普通用户只能管理自己的子域名、DNS 记录、积分、通知和资料。管理员接口同时经过会话认证与管理员中间件，前端隐藏菜单不是权限边界。

## 2. 用户与用户组

### 用户资料

用户可在个人资料页设置自定义姓名和头像。OIDC 登录只用于关联身份，不应在每次登录时用空姓名或默认头像覆盖已保存资料。管理员排查资料异常时应先确认数据库记录，再检查身份提供商声明。

### 用户组

用户组用于集中定义权限和业务额度，例如：

- 是否拥有管理员权限。
- 可认领子域名数量和记录额度。
- 与积分、审核或运营策略相关的组级配置。

删除用户组前先迁移组内用户。不要依赖前端默认组名称；以后台实际配置和数据库关联为准。

### 封禁与解封

管理员封禁用户时填写原因和可选解封时间。系统同时记录封禁开始时间；邮件和登录后的封禁页面应显示开始时间与预计解封时间。未设置结束时间表示永久封禁。

被封禁用户可查看原因并提交申诉。管理员审核通过后可执行解封；拒绝申诉不会修改原封禁期限。处置前应保留相关审核记录和证据。

## 3. DNS 提供商账号

后台可维护多个 DNS 提供商账号。凭据写入数据库前由 `ENCRYPTION_KEY` 使用 AES-256-GCM 加密；未设置加密密钥时敏感配置可能以明文存储，不适合生产环境。

建议流程：

1. 在上游创建最小权限 API Token。
2. 新增提供商账号并保存。
3. 加载可用 Zone，确认账号权限和网络连接。
4. 再将账号绑定到平台域名。

支持的提供商以后台列表为准，当前包含 Cloudflare、阿里云 DNS、DNSPod、华为云等。不同提供商对代理、TTL、根记录和 TXT 转义的支持不完全相同，平台会在统一模型内做能力适配。

## 4. 域名与子域名

### 添加域名

管理员选择 DNS 账号和上游 Zone，配置可公开认领的根域名。添加后验证：

- 公共域名列表正常展示。
- 子域名可用性检查与上游现有记录一致。
- 测试账号可以认领并创建一条记录。

### 保留前缀

保留前缀用于阻止用户认领高风险或基础设施名称，例如 `www`、`api`、`mail`、`admin`。更新保留列表不会自动释放已存在的认领，必要时由管理员单独审查。

### 域名迁移

域名迁移用于在 DNS 账号或 Zone 之间复制记录。迁移任务可查看进度、失败项、重试失败记录，并在确认完成后清理源记录。清理源端是不可逆写操作，执行前必须核对目标端记录数量和内容。

### 管理员接管

管理员可查看全部已认领子域名、释放认领或删除违规 DNS 记录。需要通知用户时填写原因，并保留审计日志。大批量操作超过阈值后进入异步任务，可在批量任务详情中查看每项结果。

## 5. DNS 记录规则

用户可维护 A、AAAA、CNAME 和 TXT：

- A 必须是合法 IPv4。
- AAAA 必须是合法 IPv6。
- CNAME 指向合法主机名，并遵守与其他记录的共存规则。
- TXT 按提供商要求处理文本和转义。

平台阻止明显重复记录和非法 CNAME 共存。所有变更请求使用幂等键；前端超时重试时复用原键，避免重复业务操作。

## 6. 积分、签到与推荐

管理员可配置积分相关规则并手工发放积分。用户端展示余额、交易流水、签到状态和推荐信息。

- 余额变化必须对应交易记录。
- 每日签到由服务端判断日期与重复请求。
- 推荐奖励按服务端规则发放，不能只依赖客户端展示。
- 手工发放应填写可审计说明，避免直接修改数据库余额。

## 7. 支付

支付配置位于后台系统设置，不再读取旧版支付环境变量。当前支持易支付与码支付兼容网关，并可独立开关支付宝、微信、QQ 等渠道。

配置项通常包括：

- 网关 URL。
- 商户 ID 与商户密钥。
- 各渠道启用状态。
- 积分产品和兑换比例。

保存外部 URL 时可能要求二次确认，防止 SSRF 或误配置。支付回调地址由公网域名生成；上线前使用小额测试订单验证创建、跳转、异步回调、同步返回和积分入账完整链路。

## 8. 自动审核与 AI 审查

### 规则审核

审核规则可按场景、记录类型、内容和动作配置。管理员可以：

- 新建、编辑、启停和删除规则。
- 使用测试面板验证规则匹配。
- 查看待处理案例、扫描历史和子域名详情。
- 单个或批量复扫。
- 恢复误处置内容或释放违规子域名。

定时扫描由后台 Worker 执行。单实例默认进程内队列，多实例应配置 Redis Streams。

### AI 模型

AI 模型配置支持 OpenAI API 兼容接口、模型名、温度、RPM 和 API Key。API Key 加密存储并脱敏展示。提示词模板支持变量插值，例如 `{{fqdn}}` 和 `{{content}}`。

AI 结果进入审查记录，可由管理员二次确认。模型结论不是前端权限判定，最终处置、封禁和恢复始终由服务端规则执行。

## 9. 通知与邮件

管理员可创建、编辑和删除站内通知，上传通知图片。用户通过列表、未读状态和 SSE 获得实时更新。

邮件管理包含 SMTP 配置、测试发送、发送记录和失败重试。生产环境需验证 TLS、发件域名、SPF/DKIM/DMARC 与网关限流。封禁邮件必须包括封禁开始时间、预计解封时间或永久封禁说明。

## 10. 品牌、SEO 与友情链接

品牌设置控制站点名称、Logo、Favicon、公告和页脚信息。Logo 可通过后台上传并由 `/api/v1/branding/logo.webp` 提供；前端会缓存公开品牌信息。

SEO 设置控制描述、关键词、公开元数据、`robots.txt`、`sitemap.xml` 和可选统计配置。上线前检查公网 URL，避免生成 `localhost` 链接。

友情链接由管理员维护，公开页面只展示启用的条目。外部 URL 应使用 HTTPS 并定期检查失效或恶意跳转。

## 11. 客户端管理

### 通讯密钥

通讯密钥只能在后台生成、轮换或作废。服务端只保存 SHA-256 哈希，明文只在生成时返回一次。管理员应立即保存到密码管理器，然后用于 GitHub Actions 客户端构建。

密钥会编译进 APK，因此它用于识别获准客户端构建，不是用户身份或管理员权限。作废密钥会使使用旧密钥的 APK 无法继续调用原生接口；应先生成新密钥并构建新客户端，再安排切换。

### 版本策略

后台可设置：

- 最新客户端版本号。
- 普通更新或强制更新。
- 中英文可读的更新公告。
- HTTPS 更新链接。

客户端每次启动调用版本接口。普通更新允许用户暂时关闭；强制更新必须阻止进入业务页面，直到存在有效下载链接。当前稳定 APK 地址为：

```text
https://hanlull.github.io/hl6/android/cc.lxii.domain/latest.apk
```

正式版本还会把同一 APK 附加到 GitHub Release。完整构建流程见[Android 客户端](android-client.md)。

## 12. 审计与日常检查

后台审计日志记录关键管理操作。建议每周检查：

- 管理员和高权限用户组变更。
- DNS 提供商 Token 与失败率。
- 域名迁移和批量任务失败项。
- 封禁、申诉和恢复记录。
- 支付异常订单与积分手工发放。
- AI 模型失败率和二次审核积压。
- Android 最新版本、强更状态和下载链接。
