# 安卓原生客户端适配规范

## 核心定位

HL6 Android 客户端必须是 Kotlin + Jetpack Compose 的纯原生应用。禁止使用 WebView、内嵌网页、H5 套壳、HTML 打包、JavaScript 运行时或任何混合渲染模式。

客户端仅负责原生 UI 渲染、Android Keystore 安全存储、调用服务端 API、展示服务端状态和响应版本更新策略。业务逻辑、数据处理、权限判断、校验、积分、DNS 操作和后台计算均由 HL6 服务端统一处理。

## 双端 UI 规则

1. Android 页面、组件、状态和交互必须以 `web/` 为唯一视觉源，保持 1:1 的颜色、间距、圆角、边框、阴影、字体层级、图标尺寸、空态、加载态、Toast 和弹窗语义。
2. `web/src/index.css` 中的设计令牌是客户端主题的来源；客户端需要同步其颜色计算、圆角、间距与状态色。
3. 网页端 UI、文案、组件、布局或交互更新时，客户端必须在同一变更中同步适配。未完成适配的变更不得合并或上线。
4. 每次迭代必须校验 UI 一致性、接口兼容性、状态一致性和功能一致性。

## 通讯密钥与网络

1. 通讯密钥只能由 HL6 管理后台生成、轮换或作废。服务端仅保存 SHA-256 哈希，生成后的明文只返回一次。
2. 客户端通过 Android Keystore 支持的加密存储保存通讯密钥和用户会话令牌，禁止使用普通 SharedPreferences、明文文件或日志输出。
3. 客户端每个 API 请求必须携带 `X-HL6-Client-Key`。服务端拒绝缺失、错误、过期或已作废的客户端密钥。
4. 管理后台作废密钥后，旧客户端必须在下次请求或启动版本检查时停止访问并提示重新安装由新密钥构建的客户端。
5. 覆盖安装含新密钥的 APK 时，客户端必须替换加密存储中的旧密钥并清除旧原生会话，强制重新登录。
6. 通讯密钥是应用级凭据，不替代 OIDC 用户认证、服务端权限校验或资源归属校验。
7. 不得将通讯密钥发送到外部头像、图标、分析或第三方资源域名；客户端外部资源必须经过经审计的服务端 API 代理或使用本地资源。

## 原生构建工作流

`.github/workflows/client-build.yml` 是唯一 Android 构建工作流。它必须支持并验证以下动态参数：通讯域名、通讯密钥、客户端名称、客户端 PNG 图标、版本号和 Android 包名。

1. 缺少任一必填参数或 Android 签名密钥时，工作流必须终止并输出明确错误。
2. 所有参数通过构建时生成的 `client/local.properties` 注入；不得在原生源码中硬编码实例域名、通讯密钥、名称、版本或包名。
3. 工作流必须产出并验证可安装的已签名 Release APK。
4. 版本号同时写入 Android `versionName` 和确定性 `versionCode`。
5. 所有构建、签名与图标资源只在工作流运行目录生成，不得提交密钥、keystore 或生成的 APK。

## 版本管控

1. 管理后台维护最新客户端版本号、强制更新开关、更新公告和 HTTPS 更新链接。
2. 客户端每次启动先请求 `/api/v1/client/version?current_version=<versionName>`；是否需要更新由服务端计算并返回，客户端不得自行比较版本号。
3. 低于最新版本时，普通更新可关闭提示；强制更新必须常驻且不能进入业务界面。
4. 更新链接、版本判断和强制策略仅由服务端返回结果控制。

## OIDC 原生适配

1. 原生应用使用系统浏览器 Custom Tabs，不使用 WebView。
2. App 必须先带通讯密钥调用 `/api/v1/auth/native/start` 创建 90 秒有效的一次性浏览器登录请求，再使用服务端返回的地址打开 Custom Tabs。通讯密钥不得出现在浏览器 URL、OIDC 参数或深链中。
3. App 使用 `hl6.<applicationId>://auth/callback` 深链接收服务端回调；该深链只携带 90 秒有效的一次性代码，不携带会话令牌。App 必须带通讯密钥调用 `/api/v1/auth/native/exchange` 交换 Bearer 会话令牌。一次性代码只能消费一次，原生会话的每个受保护请求均由服务端重新校验通讯密钥。
4. OIDC 提供商仅需继续配置服务器 HTTPS 回调地址 `/api/v1/auth/callback`，不需要向每个 Android 包名注册 OIDC 原生客户端。
5. 常见故障：深链未唤醒通常是包名与构建配置不一致；`invalid native auth code` 通常是代码过期、重复消费或客户端密钥已轮换；`invalid client key` 表示需要使用新密钥重新构建客户端。
